# Technical Debt Tracking

This file is a staging area for small technical debt items discovered during development. Items are triaged and moved to individual sprint documents in `active/` as they are prioritized.

## Directory Structure

```
technical-debt/
├── README.md                    # This file (staging area)
├── CLAUDE.md                    # AI assistant guidelines
├── active/                      # Currently being addressed
└── completed/                   # Resolved items
```

## How to Use

1. **Small items**: Add to this README under "Staging Area" below
2. **Larger items**: Create a new document in `active/`
3. **During sprint planning**: Use `/init-plan @.planning/technical-debt/active/td-##-name.md`
4. **After resolution**: Move items from active to completed
